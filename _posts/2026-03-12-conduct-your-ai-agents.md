I've recently started using [Conductor.build](https://conductor.build) and it's changed how I work with coding agents.

The core idea is simple: run multiple AI agents in parallel, each in its own Git worktree, so they can work on the same repository across different features without stepping on each other's toes.

![Conductor running multiple workspaces in parallel]({{ "/assets/images/conductor-workspaces.png" | relative_url }})

In the screenshot above, you can see three workspaces running side by side — each one tackling a different task on the same repo. No merge conflicts, no context switching, no waiting for one agent to finish before starting the next.

In fact, I used Conductor to create this very TIL blog you're reading right now.

If you've been working with coding agents and want to get more out of them, give [Conductor.build](https://conductor.build) a try. It's free to use.
